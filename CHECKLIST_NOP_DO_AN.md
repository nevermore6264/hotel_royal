# Checklist nộp đồ án – Royal Lotus Hotel

## Trước khi gửi, kiểm tra:

### 1. Mã nguồn

- [ ] Thư mục `backend/` đầy đủ (pom.xml, src, application.yml).
- [ ] Thư mục `frontend/` đầy đủ (package.json, src, vite.config.ts).
- [ ] Đã chạy `npm run build` (frontend) thành công (hoặc để GV tự build).
- [ ] File `.gitignore` có (không nộp node_modules, target, .env).

### 2. Tài liệu

- [ ] **DE_CUONG_VA_NHIEM_VU_DO_AN.md** – Đề cương + Nhiệm vụ đồ án (đã hoàn thiện).
- [ ] **README.md** – Hướng dẫn cài đặt, cấu hình, chạy backend/frontend.

### 3. Cấu hình mẫu (nếu GV cần chạy thử)

- [ ] `backend/src/main/resources/application.yml`: đã ghi rõ hoặc comment hướng dẫn đổi MySQL (url, user, password).
- [ ] Tài khoản admin mặc định: `admin` / `admin123` (được tạo khi lần đầu chạy backend).

### 4. Gửi lại

- [ ] Nén toàn bộ thư mục `hotel_royal` (trừ node_modules, backend/target, .git nếu không yêu cầu) **hoặc**
- [ ] Gửi link GitHub repo (đã push đầy đủ mã + tài liệu).

### 5. Nội dung đã hoàn thiện trong tài liệu

- **Phần I – Đề cương:** Tên đề tài, đặt vấn đề, mục tiêu, phạm vi, công nghệ, phương pháp, cấu trúc sản phẩm, kết quả dự kiến.
- **Phần II – Nhiệm vụ đồ án:** Bảng ánh xạ toàn bộ User Story (US-ADM-01 → US-SYS-04) với trạng thái Hoàn thành và mô tả triển khai Backend/Frontend; Sản phẩm nộp; Hướng dẫn chạy và kiểm tra.

---

_Chỉ cần in hoặc nộp file **DE_CUONG_VA_NHIEM_VU_DO_AN.md** kèm mã nguồn là đủ phần “đề cương và nhiệm vụ đồ án” để gửi lại._
